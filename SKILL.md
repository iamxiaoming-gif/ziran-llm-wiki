# 专题知识库 LLM Wiki 构建 Skill

> 基于 Karpathy LLM Wiki 方法论，为特定领域/专题构建深度知识库
> 参考：巴菲特投资知识库维护规则

---

## 这个 Skill 解决什么问题？

当你需要为某个领域构建**深度**知识库时，通用方法不够用。你需要：

- ✅ 明确的目录结构和分类体系
- ✅ 标准化的页面模板和写作规范
- ✅ 严格的质量控制标准和成熟度分级
- ✅ 完整的工作流和自检清单
- ✅ 可追溯的更新日志和冲突处理机制

本 Skill 提供从 0 到 1 构建专题知识库的完整框架。

---

## 核心原则

### 1. 知识处理原则

```
传统方式（❌）：先存储 → 临时查询 → 每次从零开始
LLM Wiki方式（✅）：先整理知识 → 再使用 → 持续积累
```

**核心思想：让信息只处理一次，但价值被无限放大。**

### 2. 三大铁律

| 铁律 | 说明 | 原因 |
|------|------|------|
| **原始资料只读不修改** | `00-原始资料/` 目录永远不修改 | 确保知识来源可追溯，不可污染 |
| **知识点原子化** | 一个知识点一个MD文档 | 独立、完整、可引用，形成知识网络 |
| **冲突不删除** | 发现矛盾时显式标注，不删除内容 | 保留知识演化轨迹，避免信息丢失 |

### 3. 三不三要原则

```
三不原则：
❌ 不修改原始资料（00-原始资料/ 目录只读）
❌ 不删除内容（发现矛盾时标注，不删除）
❌ 不创建重复页面（有新内容时更新旧页面）

三要原则：
✅ 要添加内部链接（用 [[知识点名称]] 语法）
✅ 要标注原文出处（用 [[00-原始资料/...]] 路径）
✅ 要更新索引和日志（每次变更都记录）
```

## 工作流规范

### 1. 摄取工作流（Ingest Workflow）

当有新资料进入 `00-原始资料/` 时：

```
┌─────────────────────────────────────────────────────────┐
│  ⚠️ 开始工作流前，请先回顾维护原则                        │
│  - 三不原则：不修改原始资料、不删除内容、不重复创建        │
│  - 三要原则：要添加内链、要标注出处、要更新索引            │
│  - 格式陷阱：数量要同步、标题要规范、入链要≥3            │
└─────────────────────────────────────────────────────────┘

Step 1: 将原始资料存入对应的 raw/ 目录
Step 2: AI Agent 读取原始资料
Step 3: AI Agent 提炼知识点，创建或更新 10-知识点库/ 中的相关页面
Step 4: AI Agent 更新知识索引
        - 20-知识索引/知识库总索引.md
        - 20-知识索引/关键词索引.md
Step 5: AI Agent 在变更日志中追加记录
        - 该知识点内嵌更新日志（第九章）
        - 30-维护记录/知识库更新日志.md
```

### 2. 查询工作流（Query Workflow）

当回答问题时：

```
┌─────────────────────────────────────────────────────────┐
│  ⚠️ 开始工作流前，请先回顾维护原则                        │
└─────────────────────────────────────────────────────────┘

Step 1: AI 读取 20-知识索引/知识库总索引.md 了解知识库结构
Step 2: 找到相关的知识点页面
Step 3: 综合页面内容给出答案
Step 4: 如果答案有价值，保存为新页面到 10-知识点库/
        ⚠️ 新建/修改页面后必须执行后续步骤！
Step 5: 更新相关索引
Step 6: 在变更日志中追加记录
Step 7: 执行 AGENTS.md 自检清单
```

### 3. 整理工作流（Lint Workflow）

建议频率：每周一次

```
常规检查：
1. 有没有知识点之间的矛盾？
2. 有没有过时的主张被新资料更新了？
3. 有没有孤立页面（没有任何链接）？
4. 有没有被提及但缺独立页面的重要概念？

格式检查：
5. 总索引数量是否与实际文件数一致？
6. 表格标题是否使用标准 ## emoji+中文 格式？
7. 新建页面是否有 ≥3 个入链？
8. 是否有 0 字节空文件残留？

内容检查：
9. 原文出处是否使用 [[双向链接]] 格式？
10. 数据（年份/金额/比例）是否准确？
11. 人物/组织名称是否与索引一致？
```

## 质量控制规范

### 1. 成熟度分级标准

| 维度 | 完整级 🟢 | 基础级 🟡 | 框架级 🔴 |
|------|----------|----------|----------|
| **章节数** | 8-9章 | 6-7章 | <6章 |
| **字数** | ≥5000字 | 2000-5000字 | <2000字 |
| **原文出处** | ≥5条 | ≥3条 | <3条 |
| **相关链接** | ≥5个 | ≥3个 | <3个 |

状态升级路径：🔴 框架级 → 🟡 基础级 → 🟢 完整级

### 2. 质量检查维度

#### 完整性检查
- 是否包含所有必需章节？
- 每个章节是否有实质内容（非空白占位）？

#### 准确性检查
- 数据是否与原文对照？
- 人名/组织名是否与索引一致？
- 原话是否标注出处？

#### 格式一致性检查
- 所有原文出处使用 [[双向链接]]
- 标题层级清晰（# → ## → ###）
- 表格前后有分割线 ---
- 统计类表格前使用 `## emoji + 中文` 标题

#### 链接健康度检查
- 每个页面出链数 ≥3
- 检查死链（目标文件是否存在）
- 检查孤立页面（是否有入链）

### 3. 成熟度分级应用

- 在总索引中用 🟢🟡🔴 标签标注每个知识点的成熟度
- 优先级：🔴框架级优先补充 → 🟡基础级完善 → 🟢完整级维护
- 补充内容时优先完善低成熟度的页面

## ⚠️ 格式陷阱（实战经验，必读）

### 陷阱1：数量占位符未回填

**现象：** 新建分类时用 `—` 占位，后续忘记更新。

```markdown
❌ | **核心概念** | — | ✅ 完成 |    ← 永远是占位符
✅ | **核心概念** | 6个 | ✅ 完成 |   ← 实际文件数
```

**规则：** 总索引中每个分类必须填写实际数字，禁止占位符。

### 陷阱2：表格标题格式不统一

```markdown
❌ **原始资料统计：**          ← 加粗文本当标题
✅ ## 📄 原始资料统计            ← 标准二级标题 + emoji
```

**规则：** 统计/汇总类表格前，必须使用标准 Markdown 标题（## 或 ###），统一 `emoji + 中文` 格式。

### 陷阱3：孤立页面遗漏

**规则：** 每新建一个知识点页面后，**必须至少在3个已有页面中添加入链**。

入链位置：
- 对应分类的"汇总"或"总览"页面
- 相关具体案例的"相关知识点"章节
- 核心概念页面的相关链接区

### 陷阱4：空文件残留

**规则：** 每次操作完成后检查目标目录是否存在空文件，发现立即删除。

## AGENTS.md 自检清单

每次新建/删除/修改知识点文件后，按顺序检查：

□ **1. 【索引同步】** 20-知识索引/知识库总索引.md
- 该分类文件数是否+1/-1？知识点总计是否同步更新？

□ **2. 【关键词同步】** 20-知识索引/关键词索引.md
- 是否有新关键词需要加入？

□ **3. 【关系图谱】** 20-知识索引/知识点关系图谱.md
- 是否有新节点需要加入？

□ **4. 【入链检查】** 新页面是否在 ≥3 个已有页面中有入链？

□ **5. 【AGENTS同步】** AGENTS.md 目录结构注释数量是否同步？

□ **6. 【内嵌日志】** 该知识点第九章「更新日志」是否已追加？

□ **7. 【集中日志】** 30-维护记录/知识库更新日志.md 是否已追加？

### 场景

### Step 1：初始化

```
请帮我在 D:/巴菲特知识库/ 初始化一个专题知识库，

- 30-维护记录/
以及 AGENTS.md 和初始索引文件。
```

### Step 2：摄取第一封股东信

将 `1956年合伙人信-原文.md` 放入 `00-原始资料/01-致股东信/`，然后：

```
请读取 00-原始资料/01-致股东信/1956年合伙人信-原文.md，
把里面的关键信息整理进 10-知识点库/：

- 在 30-维护记录/知识库更新日志.md 追加记录
```

### Step 3：AI 的输出

AI 会创建多个知识点页面：

```markdown
> 🟡 基础级 | 约2000字 | 最后更新：2026-05-27
```
