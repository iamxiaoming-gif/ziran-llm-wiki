## 一、核心原则 | Core Principles

### 1. 知识处理原则

```
传统方式（❌）：先存储 → 临时查询 → 每次从零开始
LLM Wiki方式（✅）：先整理知识 → 再使用 → 持续积累
```

**核心思想：让信息只处理一次，但价值被无限放大。**

### 2. 三大铁律

| 铁律 | 说明 | 原因 |
|------|------|------|
| **原始资料只读不修改** | `00-原始资料/` 目录永远不修改 | 确保知识来源可追溯，不可污染 |
| **知识点原子化** | 一个知识点一个MD文档 | 独立、完整、可引用，形成知识网络 |
| **冲突不删除** | 发现矛盾时显式标注，不删除内容 | 保留知识演化轨迹，避免信息丢失 |

## 二、目录结构 | Directory Structure

```
[专题名称]/
├── 00-原始资料/              ← 知识的"水源"，只读不修改
│   ├── 01-[资料分类1]/       ← [如：致股东信/论文/演讲/]
│   ├── 02-[资料分类2]/       ← [如：股东大会/访谈/播客/]
│   └── assets/               ← 图片等资源文件

├── 10-知识点库/              ← AI 整理后的知识页面（核心！）
│   ├── [分类1]/              ← [如：核心概念/投资哲学/]
│   ├── [分类2]/              ← [如：方法论/投资方法/]
│   ├── [分类3]/              ← [如：经典案例/]
│   ├── [分类4]/              ← [如：人物传记/]
│   ├── [分类5]/              ← [如：组织档案/公司档案/]
│   └── [分类6]/              ← [如：行业分析/]
│
├── 20-知识索引/              ← 知识库的"导航"
│   ├── 知识库总索引.md
│   ├── 关键词索引.md
│   └── 知识点关系图谱.md
│
├── 30-维护记录/              ← 知识库的"账本"
│   ├── 知识库更新日志.md
│   └── 冲突与错误记录.md
│
└── AGENTS.md                 ← 本维护规范
```

## 三、写作规范 | Writing Conventions

### 1. 页面模板

- 知识点页面：见 `references/知识点页面模板.md`
- 人物传记页面：见 `references/人物传记模板.md`
- 组织档案页面：见 `references/组织档案模板.md`

### 2. 通用规范

- **标题层级**：一级 `#` + 二级 `##` + 三级 `###`，不跳级
- **内部链接**：统一使用 `[[知识点名称]]` 语法
- **原文出处**：统一使用 `[[00-原始资料/分类/文件名]]` 路径
- **统计表格前**：必须使用 `## emoji + 中文` 标题格式
- **表格之间**：用 `---` 分割线分隔

### 3. 触发来源记录规范

| 触发类型 | 说明 | 示例 |
|----------|------|------|
| **用户查询** | 用户主动询问触发 | 用户查询"xxx的核心概念" |
| **Lint检查** | 定期检查发现问题 | Lint发现某页面为框架级 |
| **原始资料** | 新资料带来新信息 | 新资料新增xx信息 |
| **自发整理** | 主动完善现有内容 | 补充某案例的最新进展 |

**记录位置：**
- 主记录：该知识点的「更新日志」章节
- 备份记录：30-维护记录/知识库更新日志.md

## 四、工作流规范 | Workflow Conventions

### 1. 摄取工作流（Ingest）

```
┌─────────────────────────────────────────────────────────┐
│  ⚠️ 开始工作流前，请先回顾维护原则                        │
│  - 三不原则：不修改原始资料、不删除内容、不重复创建        │
│  - 三要原则：要添加内链、要标注出处、要更新索引            │
│  - 格式陷阱：数量要同步、标题要规范、入链要≥3            │
└─────────────────────────────────────────────────────────┘

Step 1: 将原始资料存入对应的 00-原始资料/ 目录
Step 2: AI Agent 读取原始资料
Step 3: AI Agent 提炼知识点，创建或更新 10-知识点库/ 中的相关页面
Step 4: AI Agent 更新知识索引
        - 20-知识索引/知识库总索引.md
        - 20-知识索引/关键词索引.md
Step 5: AI Agent 在变更日志中追加记录
        - 该知识点内嵌更新日志
        - 30-维护记录/知识库更新日志.md
```

### 2. 查询工作流（Query）

```
┌─────────────────────────────────────────────────────────┐
│  ⚠️ 开始工作流前，请先回顾维护原则                        │
└─────────────────────────────────────────────────────────┘

Step 1: AI 读取 20-知识索引/知识库总索引.md 了解知识库结构
Step 2: 找到相关的知识点页面
Step 3: 综合页面内容给出答案
Step 4: 如果答案有价值，保存为新页面到 10-知识点库/
        ⚠️ 新建/修改页面后必须执行后续步骤！
Step 5: 更新相关索引
Step 6: 在变更日志中追加记录
Step 7: 执行自检清单
```

### 3. 整理工作流（Lint）

建议频率：每周一次

```
常规检查：矛盾？过时？孤立？缺页？
格式检查：数量同步？标题规范？入链≥3？无空文件？
内容检查：链接格式？数据准确？名称一致？
```

## 六、维护原则 | Maintenance Principles

### 三不原则

```
❌ 不修改原始资料（00-原始资料/ 目录只读）
❌ 不删除内容（发现矛盾时标注，不删除）
❌ 不创建重复页面（有新内容时更新旧页面）
```

### 三要原则

```
✅ 要添加内部链接（用 [[知识点名称]] 语法）
✅ 要标注原文出处（用 [[00-原始资料/...]] 路径）
✅ 要更新索引和日志（每次变更都记录）
```

## 七、AGENTS.md 自检清单（每次变更后必查）

□ 1. 【索引同步】知识库总索引数量是否同步？
□ 2. 【关键词同步】是否有新关键词？
□ 3. 【关系图谱】是否有新节点？
□ 4. 【入链检查】新页面是否在 ≥3 个已有页面中有入链？
□ 5. 【AGENTS同步】目录结构注释数量是否同步？
□ 6. 【内嵌日志】该知识点更新日志是否已追加？
□ 7. 【集中日志】知识库更新日志是否已追加？

## 八、格式陷阱（Lint 经验总结）

### 陷阱1：数量占位符未回填
- 总索引中每个分类必须填写实际数字，禁止 `—` 或 `TBD`

### 陷阱2：表格标题格式不统一
- 统计类表格前必须使用 `## emoji + 中文` 标题
- 禁止用 `**文本：**` 替代标题层级

### 陷阱3：孤立页面遗漏
- 每新建页面后，必须至少在3个已有页面中添加入链

### 陷阱4：空文件残留
- 每次操作后检查空文件，发现立即删除
